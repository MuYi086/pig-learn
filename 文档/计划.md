## 一、背景与目标

基于 pig-learn 3.9.2 开源版，完整实现商业版 2026 所有功能，包括工作流、支付、报表、分布式调度、多数据库适配、跨端前端等。

## 三、实施策略

### 3.1 信心评估

**总体信心指数：85%**

理由：
- ✅ 项目架构清晰，模块化设计良好
- ✅ 已有完善的多数据库支持机制
- ✅ 公共模块抽象充分，易于扩展
- ⚠️ 报表引擎和 Uniapp 前端工作量较大
- ⚠️ 需要维护单体和微服务双模式兼容性

### 3.2 实施原则

1. **模块化扩展**：每个功能独立为 pig-common/pig-visual/pig-app 模块
2. **兼容性优先**：保持与现有代码风格、配置方式一致
3. **渐进式实现**：按优先级分阶段交付
4. **双模式支持**：同时支持微服务和单体模式

**核心 API 设计**：
```java
@RestController
@RequestMapping("/flowable/process")
public class ProcessController {
    // 流程定义管理
    @GetMapping("/definition") // 查询流程定义列表
    @PostMapping("/deploy")    // 部署流程定义（BPMN XML）
    @DeleteMapping("/{id}")    // 删除流程定义

    // 流程实例管理
    @PostMapping("/start")     // 启动流程实例
    @GetMapping("/instance")   // 查询运行中的流程实例
    @PostMapping("/complete")  // 完成当前任务节点
}

@RestController
@RequestMapping("/flowable/task")
public class TaskController {
    // 任务管理
    @GetMapping("/todo")       // 查询待办任务
    @GetMapping("/done")       // 查询已办任务
    @PostMapping("/claim")     // 认领任务
    @PostMapping("/delegate")  // 委派任务
    @PostMapping("/transfer")  // 转办任务
}
```

#### 4.4 pig-common-ijpay 支付模块

**目标**：集成 IJPay，支持微信、支付宝、银联支付

**实现步骤**：
1. 创建 `pig-common/pig-common-ijpay` 模块
2. 引入 IJPay 依赖（IJPay-WxPay、IJPay-AliPay 等）
3. 配置支付参数（支持多商户配置）
4. 实现统一下单接口
5. 实现支付回调处理
6. 实现退款接口
7. 实现订单查询接口

**数据库表设计**：
```sql
-- 支付订单表
CREATE TABLE pay_order (
    id BIGINT PRIMARY KEY,
    order_no VARCHAR(64) NOT NULL,      -- 商户订单号
    transaction_id VARCHAR(64),          -- 第三方支付流水号
    channel VARCHAR(20) NOT NULL,        -- 支付渠道：wxpay/alipay/unionpay
    amount DECIMAL(10,2) NOT NULL,       -- 支付金额
    status TINYINT DEFAULT 0,            -- 状态：0待支付 1支付成功 2支付失败
    notify_url VARCHAR(255),             -- 回调地址
    user_id BIGINT,                      -- 用户ID
    create_time DATETIME,
    update_time DATETIME
);

-- 支付配置表
CREATE TABLE pay_config (
    id BIGINT PRIMARY KEY,
    channel VARCHAR(20) NOT NULL,        -- 支付渠道
    app_id VARCHAR(64),                  -- 应用ID
    mch_id VARCHAR(64),                  -- 商户号
    api_key VARCHAR(255),                -- API密钥（加密存储）
    cert_path VARCHAR(255),              -- 证书路径
    notify_url VARCHAR(255),             -- 默认回调地址
    return_url VARCHAR(255),             -- 同步返回地址
    status TINYINT DEFAULT 1             -- 状态：0禁用 1启用
);
```

**核心 API 设计**：
```java
@RestController
@RequestMapping("/pay")
public class PayController {
    @PostMapping("/unified-order")   // 统一下单
    @PostMapping("/callback/{channel}") // 支付回调
    @PostMapping("/refund")          // 退款申请
    @GetMapping("/order/{orderNo}")  // 订单查询
}
```

**关键文件**：
- `pig-common/pig-common-ijpay/pom.xml`
- `pig-common/pig-common-ijpay/.../IJPayAutoConfiguration.java`
- `pig-common/pig-common-ijpay/.../PayService.java`
- `pig-common/pig-common-ijpay/.../PayController.java`

### 5.2 目录结构优势

1. **单一仓库管理**：前后端代码统一管理，版本同步
2. **独立构建**：前端 `pig-ui` 可独立构建部署
3. **代码生成对齐**：pig-codegen 可直接生成前端代码到 `pig-ui/src/views`
4. **开发便利**：一个仓库完成所有开发，无需切换
5. **部署灵活**：支持前后端分离部署或合并部署
