# Pig 项目启动指南

本文档说明如何手动运行 Pig 后端和前端项目。

## 环境准备

### 必需软件
- **Java**: JDK 17+
- **Maven**: 3.8+
- **Node.js**: 18+
- **MySQL**: 8.0+
- **Redis**: 6.0+
- **Nacos**: 2.3+ (微服务模式)

---

## Java 和 Maven 安装指南（Ubuntu）

如果你不熟悉 Java 环境，按以下步骤安装。

### 步骤 1：安装 Java 17

```bash
# 1. 更新软件包列表
sudo apt update

# 2. 安装 OpenJDK 17
sudo apt install openjdk-17-jdk -y

# 3. 验证安装
java -version
```

如果显示类似以下内容，说明安装成功：
```
openjdk version "17.0.8" 2023-07-18
OpenJDK Runtime Environment (build 17.0.8+7-Ubuntu-122.04)
OpenJDK 64-Bit Server VM (build 17.0.8+7-Ubuntu-122.04, mixed mode)
```

### 步骤 2：配置 JAVA_HOME 环境变量

```bash
# 1. 查找 Java 安装路径
cd /usr/lib/jvm
ls

# 应该能看到类似：java-17-openjdk-amd64

# 2. 编辑环境变量文件
sudo nano ~/.bashrc

# 3. 在文件末尾添加以下内容（根据实际目录名修改）：
export JAVA_HOME=/usr/lib/jvm/java-17-openjdk-amd64
export PATH=$PATH:$JAVA_HOME/bin

# 4. 保存并退出（Ctrl+X，然后 Y，回车）

# 5. 使配置生效
source ~/.bashrc

# 6. 验证环境变量
echo $JAVA_HOME
```

### 步骤 3：安装 Maven

```bash
# 方式一：使用 apt 安装（推荐，简单）
sudo apt install maven -y

# 验证安装
mvn -version
```

如果 apt 安装的版本低于 3.8，使用方式二手动安装：

```bash
# 方式二：手动安装最新版

# 1. 下载 Maven 3.9.x
wget https://dlcdn.apache.org/maven/maven-3/3.9.6/binaries/apache-maven-3.9.6-bin.tar.gz

# 2. 解压
tar -xzf apache-maven-3.9.6-bin.tar.gz

# 3. 移动到系统目录
sudo mv apache-maven-3.9.6 /opt/

# 4. 配置环境变量
sudo nano ~/.bashrc

# 在文件末尾添加：
export MAVEN_HOME=/opt/apache-maven-3.9.6
export PATH=$PATH:$MAVEN_HOME/bin

# 5. 保存并生效
source ~/.bashrc

# 6. 验证
mvn -version
```

### 步骤 4：验证所有环境

```bash
# Java 版本
java -version

# Maven 版本
mvn -version

# Node.js 版本（前端应该已有）
node -v

# npm 版本
npm -v
```

**期望输出：**
- Java: 17.x.x
- Maven: 3.8.x 或更高
- Node.js: 18.x 或更高
- npm: 9.x 或更高

## 可能遇到的问题

### 问题 1：command not found: java

**解决**：
```bash
# 检查是否安装成功
dpkg -l | grep openjdk

# 如果没有输出，重新安装
sudo apt install openjdk-17-jdk -y

# 如果已安装，手动添加环境变量
export JAVA_HOME=/usr/lib/jvm/java-17-openjdk-amd64
export PATH=$PATH:$JAVA_HOME/bin
```

### 问题 2：mvn 命令找不到

**解决**：
```bash
# 如果使用 apt 安装
which mvn

# 如果无输出，重新安装
sudo apt install maven -y

# 或手动添加环境变量
export MAVEN_HOME=/opt/apache-maven-3.9.6
export PATH=$PATH:$MAVEN_HOME/bin
```

### 问题 3：Maven 版本过低

**解决**：
```bash
# 卸载旧版本
sudo apt remove maven -y

# 然后使用上面的"方式二"手动安装最新版
```

## 数据库管理方案（推荐）

作为前端开发者，推荐使用 Docker 运行数据库，完全隔离、不污染系统。

### 方案 A：Docker 手动启动（最灵活）

**优点**：完全控制、数据持久化、端口自定义

```bash
# 1. 安装 Docker（如果还没有）
sudo apt update
sudo apt install docker.io -y

# 2. 创建数据目录（数据会保存在这里，不会丢失）
mkdir -p ~/docker-data/mysql
mkdir -p ~/docker-data/redis

# 3. 启动 MySQL
docker run -d \
  --name pig-mysql \
  -p 33306:3306 \
  -v ~/docker-data/mysql:/var/lib/mysql \
  -e MYSQL_ROOT_PASSWORD=root \
  -e MYSQL_DATABASE=pig \
  mysql:8.0 \
  --character-set-server=utf8mb4 \
  --collation-server=utf8mb4_unicode_ci

# 4. 启动 Redis
docker run -d \
  --name pig-redis \
  -p 36379:6379 \
  -v ~/docker-data/redis:/data \
  redis:6 \
  redis-server --appendonly yes

# 5. 等待 10 秒让 MySQL 启动完成，然后导入数据
docker exec -i pig-mysql mysql -uroot -proot < db/pig.sql
docker exec -i pig-mysql mysql -uroot -proot < db/pig_config.sql
```

**日常管理命令：**
```bash
# 查看运行状态
docker ps

# 启动
docker start pig-mysql pig-redis

# 停止
docker stop pig-mysql pig-redis

# 重启
docker restart pig-mysql

# 查看日志
docker logs pig-mysql

# 进入 MySQL 命令行
docker exec -it pig-mysql mysql -uroot -proot

# 彻底删除（数据卷还在，可重建）
docker rm pig-mysql pig-redis
```

**对应配置（application.yml）：**
```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:33306/pig?characterEncoding=utf8&useSSL=false&serverTimezone=GMT%2B8
    username: root
    password: root
  redis:
    host: localhost
    port: 36379
```

---

### 方案 B：使用项目自带的 Docker Compose（最省心）

**优点**：一键启动所有依赖、配置已写好

```bash
# 在项目根目录执行

# 1. 只启动数据库服务（-d 表示后台运行）
docker compose up -d pig-mysql pig-redis

# 2. 查看状态
docker compose ps

# 3. 查看日志
docker compose logs pig-mysql
docker compose logs pig-redis

# 4. 停止
docker compose stop pig-mysql pig-redis

# 5. 删除容器（数据会保留）
docker compose rm pig-mysql pig-redis
```

**说明：**
- MySQL 数据映射到 `~/pig-mysql-data`
- Redis 无持久化（仅开发使用）
- 端口：`33306`（MySQL）、`36379`（Redis）

**对应配置同上**，使用 `localhost:33306` 和 `localhost:36379`。

---

### 方案对比

| 方案 | 适用场景 | 数据持久化 | 复杂度 |
|------|---------|-----------|--------|
| Docker 手动 | 长期开发、需要控制 | ✅ 本地目录 | 中 |
| Docker Compose | 快速启动、临时测试 | ✅ 已配置 | 低 |
| 本地安装 MySQL | 习惯本地服务 | ✅ 系统目录 | 中 |
| H2 内存库 | 最快尝鲜、不装任何 | ❌ 重启丢失 | 低 |

**推荐：** 前端开发者用 **Docker 手动** 或 **Docker Compose**。

## 方法一：单体模式启动（推荐新手）

单体模式将所有服务打包成一个应用，无需 Nacos，启动最简单。

### 步骤 1：初始化数据库

```bash
# 1. 登录 MySQL（确保 MySQL 已启动）
mysql -u root -p

# 2. 创建数据库并导入数据
CREATE DATABASE IF NOT EXISTS pig CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE pig;
SOURCE /path/to/pig-learn/db/pig.sql;

# 3. 创建配置中心数据库
CREATE DATABASE IF NOT EXISTS pig_config CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE pig_config;
SOURCE /path/to/pig-learn/db/pig_config.sql;

# 4. 退出
EXIT;
```

### 步骤 2：配置 Redis

确保 Redis 已启动并运行（默认端口 6379）：

```bash
# 检查 Redis 是否运行
redis-cli ping
# 应返回 PONG
```

### 步骤 3：修改数据库配置

编辑 `pig-boot/src/main/resources/application.yml`：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/pig?characterEncoding=utf8&useSSL=false&serverTimezone=GMT%2B8
    username: root          # 修改为你的 MySQL 用户名
    password: yourpassword  # 修改为你的 MySQL 密码
  redis:
    host: localhost         # Redis 地址
    port: 6379              # Redis 端口
```

### 步骤 4：构建项目

```bash
# 进入项目根目录
cd /path/to/pig-learn

# 单体模式构建（-Pboot 表示单体模式）
mvn clean install -Pboot -DskipTests
```

### 步骤 5：启动后端

```bash
# 运行 pig-boot
cd pig-boot/target
java -jar pig-boot-3.9.2.jar

# 或使用 Maven 直接启动
cd pig-boot
mvn spring-boot:run -Pboot
```

启动成功后访问：
- 后端 API: http://localhost:9999
- 接口文档: http://localhost:9999/swagger-ui.html
- 登录地址: http://localhost:9999/admin

### 步骤 6：启动前端

```bash
# 进入前端目录
cd /path/to/pig-learn/pig-ui

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

启动成功后访问：
- 前端页面: http://localhost:80
- 默认账号: admin / 123456

## 方法二：微服务模式启动（完整功能）

微服务模式需要启动多个服务，包括 Nacos 注册中心。

### 步骤 1：初始化数据库

同方法一，需要导入 `pig.sql` 和 `pig_config.sql`。

### 步骤 2：启动 Nacos

方式 A：使用 Docker 启动（推荐）
```bash
docker run -d \
  --name nacos \
  -p 8848:8848 \
  -p 9848:9848 \
  -e MODE=standalone \
  -e SPRING_DATASOURCE_PLATFORM=mysql \
  -e MYSQL_SERVICE_HOST=localhost \
  -e MYSQL_SERVICE_PORT=3306 \
  -e MYSQL_SERVICE_DB_NAME=pig_config \
  -e MYSQL_SERVICE_USER=root \
  -e MYSQL_SERVICE_PASSWORD=yourpassword \
  nacos/nacos-server:v2.3.0
```

方式 B：手动下载启动
```bash
# 下载 Nacos 2.3.0
wget https://github.com/alibaba/nacos/releases/download/2.3.0/nacos-server-2.3.0.tar.gz
tar -xzf nacos-server-2.3.0.tar.gz
cd nacos/bin

# 单机模式启动
sh startup.sh -m standalone
```

Nacos 控制台：http://localhost:8848/nacos (账号: nacos / nacos)

### 步骤 3：构建项目

```bash
cd /path/to/pig-learn

# 微服务模式构建（默认）
mvn clean install -DskipTests
```

### 步骤 4：按顺序启动服务

**顺序很重要！** 必须按此顺序启动：

```bash
# 1. 启动 pig-register（Nacos 服务端，可选，如果使用外部 Nacos）
cd pig-register/target
java -jar pig-register-3.9.2.jar

# 2. 启动 pig-gateway（网关）
cd pig-gateway/target
java -jar pig-gateway-3.9.2.jar

# 3. 启动 pig-auth（认证中心）
cd pig-auth/target
java -jar pig-auth-3.9.2.jar

# 4. 启动 pig-upms-biz（用户权限服务）
cd pig-upms/pig-upms-biz/target
java -jar pig-upms-biz-3.9.2.jar
```

其他可选服务：
```bash
# 代码生成器
cd pig-visual/pig-codegen/target
java -jar pig-codegen-3.9.2.jar

# 定时任务
cd pig-visual/pig-quartz/target
java -jar pig-quartz-3.9.2.jar

# 工作流服务（新增）
cd pig-visual/pig-flowable/target
java -jar pig-flowable-3.9.2.jar

# 分布式调度中心（新增）
cd pig-visual/pig-xxl-job/target
java -jar pig-xxl-job-3.9.2.jar

# 报表服务（新增）
cd pig-visual/pig-report/target
java -jar pig-report-3.9.2.jar
```

### 步骤 5：启动前端

同方法一。

## 方法三：Docker 一键启动（最简单）

### 步骤 1：确保 Docker 已安装

```bash
docker --version
docker-compose --version
```

### 步骤 2：执行一键启动

```bash
cd /path/to/pig-learn
```

启动的服务：
- MySQL: localhost:33306
- Redis: localhost:36379
- Nacos: http://localhost:8848/nacos

```bash
cd pig-ui
npm install
npm run dev
```

## 常见问题

### 1. 端口冲突

如果端口被占用，修改对应服务的 `application.yml`：

```yaml
server:
  port: 新端口号
```

### 2. 数据库连接失败

检查：
- MySQL 是否已启动
- 用户名密码是否正确
- 数据库 `pig` 和 `pig_config` 是否已创建

### 3. Nacos 连接失败

检查：
- Nacos 是否已启动
- `pig-register` 的端口 8848 是否可访问
- 配置文件中的 `NACOS_HOST` 和 `NACOS_PORT` 是否正确

### 4. 前端无法连接后端

检查：
- `pig-ui/vite.config.ts` 中的代理配置
- 后端服务是否正常启动
- 浏览器控制台是否有跨域错误

### 5. 代码格式化错误

提交前必须执行：
```bash
mvn spring-javaformat:apply
```

或在 IDEA 中安装 spring-javaformat 插件自动格式化。
